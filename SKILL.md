# 岗位调研报告生成器

## 概述

本 skill 通过多轮联网搜索调研指定公司的指定岗位，收集公司信息、JD、薪资、技能要求等数据，生成标准 A4 版式的中文 PDF 调研报告。

## 工作流

### 第一步：解析用户输入

从用户查询中提取：
- **公司名称**：必需
- **岗位名称**：必需
- **指定搜索网站（可选）**：如果用户指定了特定网站（如 BOSS 直聘、猎聘等），所有搜索必须严格限定在该网站范围内；否则由 Codex 自行判断最优搜索源

### 第二步：分轮搜索收集数据

按以下顺序进行联网搜索，每轮搜索结果需保存为结构化文本：

**第 1 轮 — 公司概况**
搜索公司信息，覆盖四个维度：
- 行业定位（所属赛道、主营业务）
- 公司规模（成立时间、员工人数、分城市布局）
- 发展阶段（融资轮次、估值、上市状态，如天使轮/A 轮/D 轮/已上市等）
- 企业文化（价值观、工作氛围、特色福利）

**第 2 轮 — 岗位职责（JD 拆解）**
搜索该岗位的详细 JD，覆盖：
- 岗位概述
- 具体职责列表（逐条）
- 任职要求列表（学历、经验等硬性条件）

**第 3 轮 — 薪资范围**
搜索该岗位的薪资数据，按三个维度组织：
- 按城市划分（至少列出 3 个主要城市）
- 按经验年限划分（初级/中级/高级）
- 按职级划分（专员/经理/高级/总监等）

**第 4 轮 — 技能要求**
搜索该岗位所需的技能，分为：
- 硬技能（技术工具、编程语言、行业知识等）
- 软技能（沟通、逻辑、管理能力等）

### 第三步：构造 JSON 数据

将收集到的数据整理成以下 JSON 格式保存为临时文件。注意确保所有中文字段正确编码。

```json
{
  "company": "公司全称",
  "position": "岗位名称",
  "report_date": "2026年6月",
  "gen_date": "2026-06-18",
  "company_info": {
    "industry": [
      "行业定位描述，一至两段",
    ],
    "scale": [
      "公司规模描述，一段即可",
    ],
    "development_stage": [
      "发展阶段描述，如融资轮次/上市状态等",
    ],
    "culture": [
      "企业文化描述，一至两段",
    ]
  },
  "job_description": {
    "overview": "岗位概述一句话",
    "duties": [
      "具体职责 1",
      "具体职责 2"
    ],
    "requirements": [
      "任职要求 1",
      "任职要求 2"
    ]
  },
  "salary": {
    "by_city": [
      {"city": "城市名", "range": "薪资区间", "note": "补充说明"}
    ],
    "by_experience": [
      {"level": "经验年限", "range": "薪资区间", "note": "补充说明"}
    ],
    "by_rank": [
      {"rank": "职级名称", "range": "薪资区间", "note": "补充说明"}
    ]
  },
  "skills": {
    "hard_skills": [
      "硬技能 1",
      "硬技能 2"
    ],
    "soft_skills": [
      "软技能 1",
      "软技能 2"
    ]
  },
  "references": [
    "[序号] 来源名称。文章标题[EB/OL]. URL, 访问日期。"
  ]
}
```

**JSON 构造指南：**

- `company` 和 `position`：直接使用从用户输入解析的值
- `report_date`：格式为 `YYYY年M月`
- `gen_date`：当天日期，格式 `YYYY-MM-DD`
- 所有数组字段至少包含 1 条内容，主要内容字段（如 industry、duties）建议 3-5 条
- 薪资数据每个维度至少 3 条记录
- 如果某维度未搜索到数据，使用空数组 `[]` 而非省略字段
- 参考文献格式遵循 GB/T 7714 标准

### 第四步：生成 PDF

调用 `scripts/generate_report.py` 脚本：

```bash
python scripts/generate_report.py <input.json> <output.pdf>
```

**文件命名规则：** `{公司名}_{岗位名}_调研报告.pdf`

输出文件保存在用户当前工作目录下。

### 第五步：展示结果

PDF 生成成功后，向用户展示：
- 输出文件路径
- PDF 大小
- 简要概括报告包含的内容模块

## 注意事项

1. **数据时效性**：薪资数据可能因市场波动而变化，报告中需注明调研日期
2. **信息可靠性**：优先采用招聘平台官方数据（BOSS 直聘、猎聘、拉勾网等），次选行业报告或媒体报道
3. **引用标注**：所有搜索来源必须在 `references` 字段中记录
4. **无用信息过滤**：搜索结果中可能包含与目标公司/岗位无关的内容，需筛选后填入
5. **语言一致**：报告正文全部使用中文，薪资单位使用 K（千元/月）
